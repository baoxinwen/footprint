# 旅行足迹地图 — 测试计划

> 版本：v1.0 | 日期：2026-06-01

---

## 1. 测试范围

### 1.1 功能模块清单

| 编号 | 模块 | 子功能 | 接口数 |
|------|------|--------|--------|
| M01 | 账号系统 | 注册、登录、退出、修改密码 | 3 |
| M02 | 旅行管理 | 列表、新建、编辑、删除、详情、地点增删改、排序 | 8 |
| M03 | 照片管理 | 上传、查看原图/缩略图、删除、按地点列表 | 5 |
| M04 | 导入导出 | JSON导出、Markdown导出、JSON导入 | 3 |
| M05 | 分享 | 创建分享、查看分享（含过期处理） | 2 |
| M06 | 统计分析 | 概览、年度、月度、城市排行、地图统计、城市标记、路线 | 7 |
| M07 | 时间线 | 按年月分组展示 | 1 |
| M08 | 安全 | JWT认证、数据隔离、频率限制、密码加密 | - |

### 1.2 不在测试范围内

- 高德地图 JS API 的实际调用（需真实 Key，属于集成/E2E 层）
- 前端 UI 视觉还原（属于人工测试）
- 性能/压力测试（个人工具，低优先级）

---

## 2. 测试策略

### 2.1 单元测试（test/unit/）

**目标：** 验证独立函数和工具模块的正确性。

| 测试对象 | 文件 | 框架 |
|---------|------|------|
| 密码加密/验证 | test_security.py | pytest |
| JWT 生成/解析 | test_security.py | pytest |
| 图片验证/保存/删除 | test_image.py | pytest |
| 频率限制逻辑 | test_rate_limit.py | pytest |
| Pydantic Schema 校验 | test_schemas.py | pytest |

### 2.2 集成测试（test/integration/）

**目标：** 验证 API 接口的请求-响应-数据库交互。

使用 FastAPI 的 `TestClient` + 内存 SQLite，覆盖所有 36+ 个接口。

| 测试文件 | 覆盖模块 |
|---------|---------|
| test_auth.py | M01 账号系统 |
| test_trips.py | M02 旅行管理 |
| test_photos.py | M03 照片管理 |
| test_export_import.py | M04 导入导出 |
| test_shares.py | M05 分享 |
| test_stats.py | M06 统计分析 |
| test_timeline.py | M07 时间线 |

### 2.3 端到端测试（test/e2e/）

**目标：** 模拟用户完整操作流程。

| 用例 | 描述 |
|------|------|
| test_e2e_flow.py | 注册→登录→创建旅行→添加地点→上传照片→查看→导出→分享→退出 |

## 3. 测试环境

| 项目 | 说明 |
|------|------|
| Python | 3.12+ |
| 测试框架 | pytest + pytest-html |
| HTTP 客户端 | httpx (FastAPI TestClient) |
| 数据库 | SQLite 内存模式（`:memory:`） |
| 文件存储 | 临时目录（`tempfile.mkdtemp`） |
| 运行方式 | `cd test && pytest` |

## 6. 风险说明

| 风险 | 影响 | 应对 |
|------|------|------|
| 高德 API 需要真实 Key | 地图相关功能无法自动测试 | Mock AMap JS API |
| 照片上传需要文件 | 测试需生成测试图片 | 使用 Pillow 程序化生成 |
| SQLite 并发限制 | 并发测试可能不稳定 | 使用内存数据库，串行执行 |
