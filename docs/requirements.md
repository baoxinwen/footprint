# 旅行足迹地图 — 产品需求文档（PRD）

> 版本：v1.1
> 最后更新：2026-06-01

---

## 一、产品概述

### 1.1 产品定位

一款私人的旅行足迹记录工具，帮助用户在地图上标记去过的城市和景点，上传旅行照片，撰写旅行游记，并以时间线和统计图表回顾自己的旅行经历。

### 1.2 技术栈

| 层级 | 技术选型 |
|------|---------|
| 前端 | Vue 3 + TypeScript + Vite + Element Plus |
| 后端 | Python FastAPI + SQLAlchemy + Pillow |
| 数据库 | SQLite |
| 地图服务 | 高德地图 JS API v2.0 |
| 部署 | Docker Compose（frontend: nginx, backend: uvicorn） |

### 1.3 部署架构

```
docker-compose.yml
├── frontend (nginx)    → 静态文件托管 + 反向代理 /api → backend
└── backend (uvicorn)   → FastAPI 应用
    ├── /data/footprint.db     → SQLite 数据库（volume 持久化）
    └── /data/uploads/         → 照片存储（volume 持久化）
```

### 1.4 坐标系

使用高德坐标系（GCJ-02）。

## 三、账号系统

### 3.1 注册

**功能描述：** 用户输入用户名和密码创建账号。

**字段规则：**

| 字段 | 规则 |
|------|------|
| 用户名 | 仅允许字母、数字、下划线，3-50 个字符，不可重复 |
| 密码 | 至少 6 位，无复杂度要求 |

**业务规则：**
- 注册成功后自动切换到登录 Tab，显示成功提示
- 用户名已存在时提示"该用户名已被占用"

**接口频率限制：** 同一 IP 每小时最多注册 3 个账号

### 3.2 登录

**功能描述：** 用户输入用户名和密码登录。

**业务规则：**
- 登录成功后返回 JWT token，前端存储于 localStorage
- token 有效期 24 小时，过期后跳转登录页，不做无感刷新
- 支持多设备同时登录
- 登录成功跳转至地图首页 `/`

**接口频率限制：** 同一用户名连续 5 次失败后锁定 15 分钟

### 3.3 退出

**功能描述：** 导航栏提供退出按钮。

**业务规则：**
- 点击后清除 localStorage 中的 token
- 跳转至 `/login`

### 3.4 修改密码

**功能描述：** 设置页面可修改密码。

**字段规则：**

| 字段 | 规则 |
|------|------|
| 当前密码 | 必须与当前密码一致才能修改 |
| 新密码 | 至少 6 位 |
| 确认密码 | 必须与新密码一致 |

**业务规则：**
- 修改成功后清除 token，跳转至登录页，提示"密码修改成功，请重新登录"
- 旧密码立即失效
- 修改成功后 1 小时内不允许再次修改

## 四、地图首页

### 4.1 地图展示

**功能描述：** 首页是一张可交互的高德地图，默认视野显示中国全境。用户可缩放和拖动地图。

**空状态：** 无旅行记录时，地图中央展示引导文案"标记你的第一个旅行目的地"，附带新建旅行按钮。

### 4.2 城市标记

**功能描述：** 用户去过的所有城市在地图上显示标记点。

**业务规则：**
- 城市到访次数 = 包含该城市的旅行数量（一次旅行中该城市出现多个地点只算 1 次）
- 不同到访频次使用不同颜色/大小的标记区分
- 点击城市标记 → 跳转至旅行列表页，自动筛选该城市的旅行

### 4.3 行程路线

**功能描述：** 包含多个地点的旅行在地图上用彩色虚线连接各地点，形成行程路线。

**业务规则：**
- 地图默认仅显示城市标记点，不显示路线连线
- 用户可通过筛选器（旅行列表下拉）选择某次旅行，单独查看其路线
- 不同旅行使用不同颜色区分
- 连线顺序跟随旅行详情页中地点的排列顺序
- 仅展示有多地点的旅行路线

### 4.4 统计概览

**功能描述：** 地图左上角显示统计面板（可折叠）。

**展示指标：**

| 指标 | 计算方式 |
|------|---------|
| 旅行总次数 | 用户所有旅行数量 |
| 到访地点数 | 所有旅行包含的地点总数（去重） |
| 到访城市数 | 所有旅行涉及的去重城市数 |
| 到访省份数 | 所有旅行涉及的去重省份数 |

**移动端适配：** 统计面板默认折叠收起，点击展开。

### 4.5 地图样式

**功能描述：** 右上角提供地图样式切换按钮（如标准、卫星、暗色等）。

**业务规则：** 用户选择的样式持久化存储（localStorage），下次打开自动应用。

## 五、旅行管理

### 5.1 旅行列表

**功能描述：** 以卡片列表形式展示所有旅行记录。

**卡片信息：** 标题、日期（开始 ~ 结束）、描述摘要、城市标签、地点数量。

**排序功能：**

| 排序方式 | 默认 | 说明 |
|---------|------|------|
| 按日期 | 默认，降序 | 开始日期 |
| 按名称 | - | 标题字母序 |
| 按地点数量 | - | 地点数量 |

三种排序方式均可切换升降序。

**搜索功能：**
- 顶部搜索框，按标题、描述、城市模糊搜索
- 搜索时显示匹配数量（如"找到 3 条旅行记录"）

**分页：** 默认每页 20 条，支持无限滚动加载。

**空状态：** 旅行列表为空时展示引导文案"还没有旅行记录，去创建第一次旅行吧"，附带新建旅行按钮。

### 5.2 新建旅行（独立页面 `/trips/new`）

**功能描述：** 用户填写旅行信息、添加地点、上传照片、编写游记。

**表单字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 标题 | 文本 | 是 | 旅行标题 |
| 描述 | 文本域 | 否 | 旅行简述 |
| 开始日期 | 日期选择器 | 是 | - |
| 结束日期 | 日期选择器 | 是 | 须 ≥ 开始日期 |

**地点搜索：**
- 输入景点名称，全国范围搜索高德 POI
- 返回最多 10 个匹配结果，显示名称和地址
- 点击结果即添加，地图自动跳转到该位置
- 地点信息自动从高德结果解析：名称、地址、经纬度、所属城市、所属省份

**地点管理：**
- 已添加的地点显示在列表中，可删除
- 每个地点可展开，展开后可上传照片和编写游记（Markdown 格式，双栏编辑）
- 新地点默认添加到列表末尾

**照片上传：**
- 用户填写标题和日期后，系统自动保存旅行草稿（后台创建 trip 记录），使照片上传成为可能
- 草稿保存后，各地点可正常上传照片
- 点击"保存"按钮完成创建，跳转至旅行详情页
- 若用户未填写必填字段就添加地点并上传照片，系统提示"请先填写标题和日期"

**保存规则：**
- 必须填写标题、开始日期、结束日期才能最终保存
- 可以不添加地点直接保存

### 5.3 编辑旅行（独立页面 `/trips/:id/edit`）

**功能描述：** 可修改旅行的所有信息。

**业务规则：**
- 可修改标题、描述、日期
- 可新增或删除地点
- 可编辑各地点的游记（支持从已上传照片中选择插入图片）
- 编辑页预填充原有数据
- 删除地点时标记为待删除，不立即执行；用户点击保存后才真正删除地点及其关联照片
- 放弃编辑（未保存直接离开）时不执行任何删除操作

### 5.4 删除旅行

**功能描述：** 删除一次旅行记录。

**业务规则：**
- 删除前弹出确认对话框，提示"删除后将同时清除该旅行下的所有地点和照片，且不可恢复"
- 确认后删除旅行及其关联的所有地点和照片
- 删除后统计数据立即刷新

### 5.5 旅行详情（`/trips/:id`）

**功能描述：** 展示一次旅行的完整信息。

**展示内容：**
- 标题、日期（开始 ~ 结束）、描述
- 所有地点按排列顺序展示，每个地点以可折叠卡片呈现

**地点卡片（折叠状态）：** 序号、名称、城市 · 省份、照片数量、游记摘要

**地点卡片（展开状态）：** 点击卡片展开，完整展示该地点的内容：
- 照片缩略图网格（点击可全屏查看，详见 6.2）
- 游记内容（Markdown 渲染后展示，支持深色模式）
- 上传照片按钮（详情页直接可用，无需进入编辑态）

**操作按钮：** 编辑、导出（JSON / Markdown）、分享、删除

### 5.6 地点排序

**功能描述：** 详情页的地点列表支持拖拽排序。

**业务规则：**
- 桌面端：鼠标拖拽
- 移动端：长按拖拽
- 拖拽一次立即保存顺序
- 排序顺序影响地图行程路线的连线顺序

### 5.7 导出

**功能描述：** 将单次旅行记录导出，支持两种格式。

**格式一：JSON（用于数据备份和导入）**

导出为 `.json` 文件，照片不包含在内。

```json
{
  "title": "北京三日游",
  "description": "国庆假期北京之旅",
  "startDate": "2025-10-01",
  "endDate": "2025-10-03",
  "locations": [
    {
      "name": "故宫博物院",
      "address": "北京市东城区景山前街4号",
      "longitude": 116.397128,
      "latitude": 39.916527,
      "city": "北京",
      "province": "北京",
      "note": "## 游记\n\n故宫真的很大..."
    }
  ]
}
```

**格式二：Markdown（用于阅读和分享）**

导出为 `.zip` 压缩包，结构如下：

```
北京三日游.zip
├── 北京三日游.md          ← Markdown 文档
└── photos/                ← 照片文件夹
    ├── 故宫博物院/
    │   ├── photo_001.jpg
    │   └── photo_002.jpg
    └── 长城/
        └── photo_003.jpg
```

**Markdown 文件内容结构：**
- 一级标题：旅行标题
- 日期和描述
- 各地点按顺序依次展示：地点名称、城市 · 省份、游记内容（Markdown 原文渲染）
- 游记中的图片使用相对路径引用（如 `photos/故宫博物院/photo_001.jpg`）
- 无游记的地点仅展示地点信息

**导出触发：** 点击导出按钮后弹出选择对话框，用户选择 JSON 或 Markdown 格式。

### 5.8 导入

**功能描述：** 支持从 JSON 文件批量导入旅行数据。

**业务规则：**
- JSON 格式与导出格式一致
- 导入的数据归属于当前用户
- 照片字段留空，用户在详情页手动补传
- 重复数据不做去重，全部新增

### 5.9 分享

**功能描述：** 点击分享按钮生成分享链接。

**业务规则：**
- 使用随机 UUID 作为 token，生成链接格式：`http://localhost:5173/share/{uuid}`
- 链接自动复制到剪贴板
- 链接 30 天后失效
- 分享页面复用旅行详情页布局，隐藏操作按钮（编辑、删除、导出、分享）
- 无需登录即可查看
- 访问过期链接时跳转至提示页面，显示"该分享链接已过期"

## 六、照片管理

> 照片依附于地点，所有照片操作均在旅行详情页中展开对应地点卡片后进行。

### 6.1 上传照片

**功能描述：** 在旅行详情页或新建旅行页面，展开地点卡片后显示上传按钮。

**支持格式：** JPG、PNG、GIF、WebP

**限制规则：**

| 规则 | 值 |
|------|-----|
| 单张大小限制 | 10MB |
| 单次上传数量 | 无限制 |

**业务规则：**
- 上传时显示进度条
- 系统验证文件格式合法性（检查文件头，非仅扩展名）
- 自动生成缩略图：保持原图比例，宽度 300px
- GIF 文件截取第一帧作为静态缩略图
- 文件重命名存储，防止路径猜测

### 6.2 查看照片

**功能描述：** 照片以缩略图网格展示。

**业务规则：**
- 点击任意照片打开全屏查看器
- 桌面端：左右箭头键或屏幕两侧按钮切换，ESC 关闭
- 移动端：左右滑动手势切换，下滑关闭

### 6.3 删除照片

**功能描述：** 鼠标悬停（移动端长按）在照片上出现删除按钮。

**业务规则：**
- 点击删除按钮后弹出确认对话框
- 确认后同时移除原图和缩略图

**功能描述：** 每个地点可撰写一段游记，支持 Markdown 语法。

## 九、统计分析

### 9.1 数据概览

**空状态：** 无旅行记录时，数据卡片均显示为 0，图表区域展示引导文案"记录你的第一次旅行，这里将展示统计数据"。

展示四项核心数据卡片：

| 指标 | 计算方式 |
|------|---------|
| 旅行次数 | 所有旅行总数 |
| 到访城市数 | 去重城市数 |
| 到访省份数 | 去重省份数 |
| 累计旅行天数 | 每次旅行天数 = 结束日期 - 开始日期 + 1，同一天算 1 天，所有旅行天数求和 |

### 9.2 年度统计

水平柱状图，展示每年的旅行次数，按年份倒序排列。

### 9.3 月度统计

垂直柱状图，展示 1-12 月的旅行分布（跨所有年份），帮助用户发现常出行月份。

### 9.4 城市排行榜

到访次数最多的前 10 个城市，前三名高亮显示。

**到访次数计算：** 同城市到访规则与地图标记一致（一次旅行算 1 次）。

## 十一、安全与隐私

### 11.1 认证方案

- 使用 JWT（JSON Web Token）认证
- 登录成功返回 token，前端存储于 localStorage
- 每次请求通过 `Authorization: Bearer {token}` Header 携带
- token 有效期 24 小时，过期后跳转登录页
- 支持多设备同时登录

### 11.2 数据隔离

每个用户只能看到自己的旅行数据。所有接口（除登录、注册、分享页面）均验证 token。

### 11.3 密码安全

密码使用 bcrypt 等算法加密存储，不以明文保存。

### 11.4 文件安全

- 上传照片验证文件头（magic bytes），不仅检查扩展名
- 文件重命名后存储（UUID 命名），防止路径猜测

### 11.5 接口频率限制（应用层实现）

| 接口 | 限制规则 |
|------|---------|
| 注册 | 同一 IP 每小时最多 3 次 |
| 登录 | 同一用户名连续 5 次失败后锁定 15 分钟 |
| 修改密码 | 成功后 1 小时内不允许再次修改 |

## 十三、Docker Compose 部署

```yaml
version: '3.8'

services:
  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

  backend:
    build: ./backend
    ports:
      - "8000:8000"
    volumes:
      - ./data/db:/app/data        # SQLite 数据库
      - ./data/uploads:/app/uploads # 照片存储
    environment:
      - JWT_SECRET=your-secret-key
```

**nginx 反向代理配置：**
- `/api/*` 转发至 `backend:8000`
- 其他路径返回前端静态文件
